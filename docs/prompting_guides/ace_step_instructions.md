# MoodScape — ACE-Step 1.5 Prompt Instructions

This document is the authoritative reference for writing background music prompts for MoodScape when using the **ACE-Step 1.5** music engine. It is intended to be given to an LLM to generate prompts that feed directly into the Gradio UI's **Music Style Prompt** field.

---

## How the App Uses Your Prompt

Your prompt is **not passed directly to ACE-Step**. Before generation, `core/pipeline.py` calls `_enhance_acestep_prompt()`, which delegates to `AceStepEngine._enhance_prompt()`. This function applies the **MESA framework** to build a **(caption, lyrics)** pair:

### Caption construction (`caption` field)

The engine prepends a set of **base tags** — filtered to avoid duplicating any words you already provided — then appends your raw prompt verbatim, then appends an instrumental constraint:

```
[Filtered base tags], [YOUR PROMPT VERBATIM], no vocals, instrumental
```

The base tags are:
```
ambient, meditation, calm, peaceful, warm, spacious, soft dynamics, gentle, soothing, high fidelity, studio quality, clean production
```

Any base tag whose significant words already appear in your prompt is suppressed. For example, if you write `"warm analog pads"`, the word `warm` is detected and the `warm` base tag is skipped.

**Critical implication:** The base tags already supply core mood and quality anchors. Your prompt's job is to provide **specific instrument, texture, or tonal character** that sits between these anchors and the instrumental constraint.

### Lyrics construction (duration-aware structural tags)

Even for instrumental music, ACE-Step uses section tags from the lyrics field to guide arc and structure. These are auto-generated by the engine based on the target duration — you do not write them. They scale as follows:

| Duration | Structure generated |
|----------|---------------------|
| ≤ 90s | `[Instrumental]`, Intro, Verse, Outro |
| 90s – 300s | `[Instrumental]`, Intro, Verse, Bridge, Verse, Outro |
| > 300s | `[Instrumental]`, Intro, Verse, Bridge, Interlude, Verse, Bridge, Outro |

**You can append extra structural hints** via the `lyrics` parameter if needed (see [Story Mode](#story-mode-multi-stage-generation)), but for standard single-prompt use the auto-generated structural tags are sufficient and optimal.

## What the App Actually Generates

### Model & backend

ACE-Step 1.5 uses a **two-component architecture** on Apple Silicon (MLX):

| Component | Role |
|-----------|------|
| **LLM** — `acestep-5Hz-lm-4B` (4B variant) | **Primary Planner:** 4B Qwen3 model (falls back to 1.7B if 4B is missing). Chain-of-Thought planning: reads your caption, plans tempo, key, section structure, and 5Hz semantic audio codes |
| **DiT** — `acestep-v15-sft` (~2B params) | Renders 48 kHz stereo audio from the LM plan via iterative diffusion (50 steps, SFT config) |

**Key insight:** The LM planner's interpretation of your caption determines the final output quality. The move to the **4B model** significantly improves long-form motif retention (remembers themes across 10+ min) and comprehension of complex, layered prompts. A clear, coherent caption produces a coherent plan.

### Fixed generation parameters (not user-adjustable)

| Parameter | Value | Effect |
|-----------|-------|--------|
| `guidance_scale` | `5.0` | SFT optimal range 5–7; balances prompt adherence with smooth ambient texture |
| `inference_steps` | `50` (SFT) / `8` (Turbo) | Max for SFT without error accumulation |
| `lm_temperature` | `0.85` | Richer tonal palettes while maintaining structural coherence |
| `use_adg` | `True` | Adaptive Dual Guidance — reduces spectral noise without additional inference cost |
| `instrumental` | `True` | **Always forced** — vocals are categorically disabled |
| `thinking` | `True` | Enables Chain-of-Thought reasoning in the LM planner |
| `use_cot_metas` | `True` | CoT planning includes musical metadata (tempo, key, timbre tags) |
| `enable_normalization` | `True` | VAE output is normalized before returning |

### Generation quality (UI control)

The **"Generation Quality"** radio button appears in the Gradio UI when ACE-Step 1.5 is selected:

| UI Label | Internal value | Steps | Use case |
|----------|---------------|-------|----------|
| **Studio (SFT / 60-step)** | `sft` | 50 | Production — highest fidelity. Default. |
| **Draft (Turbo / 8-step)** | `turbo` | 8 | Fast preview — auditions prompt ideas without committing to a full generation |

> **Recommendation:** Always use **Studio** for final output. Use Draft only to quickly verify prompt direction.

### Duration-based generation pipeline

The engine automatically selects a strategy based on target duration:

| Duration | Strategy |
|----------|----------|
| ≤ 90s | Single ACE-Step inference call. Simple, reliable. |
| > 90s | **Three-phase infinite pipeline** (genesis → cover continuation → boundary smoothing). All audio processed at native 48 kHz stereo before final downmix. |

For the three-phase pipeline:
- **Phase 1 (Genesis):** 60s anchor segment via `text2music`. Sets the harmonic DNA and timbre palette.
- **Phase 2 (Continuation):** Subsequent 60s segments via `cover` task, with `audio_cover_strength` decaying from 0.85 → 0.70. The cover task inherits harmonic structure while allowing tonal evolution.
- **Phase 3 (Boundary Smoothing):** 5-second repaint windows at each segment seam to eliminate audible joins.

**Your prompt governs the entire track** — the same enhanced caption is used for all phases. A prompt that clearly establishes a tonal world (rather than listing multiple contrasting moods) produces the most coherent long-form output.

### Post-processing applied to ACE-Step output

After generation, the audio passes through two sequential chains:

**1. Engine postprocess** (in `_postprocess()`):
```
48 kHz stereo tensor
  → Stereo-to-mono (channel average)
  → Resample 48 kHz → 24 kHz (Kaiser-windowed sinc, rolloff=0.9475)
  → Peak normalize to -1 dBFS
  → Safety clip to [-1, 1]
```

**2. ACE-Step music FX chain** (`make_acestep_music_chain()` in `core/audio_processor.py`):
```
HighpassFilter(60 Hz)                    ← removes diffusion noise rumble below 60 Hz
LowShelfFilter(200 Hz, +2.0 dB)         ← enveloping warmth (conservative at -14 LUFS premix)
PeakFilter(4000 Hz, -1.5 dB, Q=0.8)    ← softens upper-mids to sit behind narration
HighShelfFilter(10000 Hz, -1.0 dB)      ← smooths the digital edge
Compressor(-20 dB, 2.5:1, 80ms, 800ms) ← tighter macro-control, meditative release
Limiter(-0.5 dBFS)
```

**3. Master chain** applied at export:
```
HighpassFilter(35 Hz) → Gain(-3 dB) → Compressor(2:1 glue) → Limiter(-1.0 dBFS)
```

**What this means for prompts:**
- The **60 Hz highpass** removes sub-bass rumble — you do not need to avoid sub-bass in your prompt; the FX chain handles it.
- The **+2.0 dB low-shelf at 200 Hz** already adds warmth to all ACE-Step output. You can freely include `warm` in your prompt and the effect will compound positively.
- The **-1.5 dB cut at 4 kHz** creates a spectral pocket for narration. Prompts emphasizing upper-mid presence instruments (bright piano, metallic overtones) will be attenuated.
- The **-1.0 dB shelf at 10 kHz** tames digital edge. Do not add `bright` or `sparkle` descriptors expecting them to survive at full intensity — they will be reduced.
- The target pre-mix loudness is **-14 LUFS** (matching the final streaming standard). The music enters the mixer at this level, then the `music_volume_db=-17 dB` offset is applied, followed by the configured duck amount during speech.

## Gradio UI Controls Reference

When **ACE-Step 1.5** is selected in the Background Music Model dropdown, the following controls appear:

| Control | Location | Range / Options | Default | Notes |
|---------|----------|-----------------|---------|-------|
| **Music Style Prompt** | Main left panel | Free text | See below | The primary input this guide covers |
| **Generation Quality** | Right panel (visible) | Studio / Draft | Studio | Use Studio for production |
| **BPM** | ACE-Step Metadata accordion | 40–100 | 50 | Sets the LM planner's tempo target |
| **Musical Key** | ACE-Step Metadata accordion | Auto + 24 keys/modes | Auto | Constrains harmonic planning to a key |
| **Reference Audio** | Main left panel | File upload | None | Guides timbre/acoustic style (not melody) |

### BPM guidance

BPM is passed directly to the LM planner as a metadata constraint — it is **not included in the caption**. Including BPM numbers in the caption text is redundant and wastes tokens.

| BPM range | Effect |
|-----------|--------|
| 40–50 | Extremely slow, deep drone, theta-wave territory |
| 50–65 | Standard meditation — slow evolution, minimal pulse |
| 65–80 | Gentle movement, slightly more melodic flow |
| 80–100 | Structured musical feel — not ideal for deep meditation |

> **Default (50 BPM):** Optimal for ambient meditation. Do not raise above 70 unless the session explicitly calls for a lighter, morning-style feel.

### Musical Key guidance

Key is passed to the LM planner as a metadata constraint — **do not include key names in the caption text**.

| Key choice | Effect |
|------------|--------|
| **Auto** | LM planner chooses based on prompt mood. Usually selects warm minor keys for ambient material. Recommended for most use cases. |
| Minor keys (e.g. A Minor, D Minor) | Introspective, grounding, meditative. Preferred for deep sleep and body scan. |
| Major keys (e.g. C Major, G Major) | Open, hopeful, spacious. Better for morning, awakening, visualization. |
| Flat keys (Eb, Ab, Bb) | Warm, rounded timbre — suits drone-heavy pads and bowls. |

### Reference Audio

When a reference audio file is uploaded:
- For ACE-Step: guides **timbre and acoustic style** (the "colour" of the sound — room character, instrument family warmth). It does **not** guide melody or chord progression the way MusicGen's melody conditioning does.
- Provided as a temp `.wav` file path to the `reference_audio` parameter of `GenerationParams`.
- The influence is subtle — the LM planner still drives structure; the reference audio shapes the timbral space.

## Prompt Structure

### Formula

```
[Primary Sound Source] + [Texture/Character] + [Tonal/Atmospheric Quality]
```

Keep to **2–4 descriptive clauses**, comma-separated. Each clause should be 3–6 words. The base tags already cover mood and quality anchors — **your prompt provides specificity**.

### What to include: Sound Sources

These instruments and textures produce excellent meditation ambient material via the LM planner:

| Term | LM Planner Output |
|------|------------------|
| `warm analog synth pads` | Slow attack synth texture, thick harmonic content |
| `tibetan singing bowls` | Metallic resonant tones, long bell-like decay |
| `crystal singing bowls` | Higher, glassier bowl tones |
| `soft piano` | Warm, gently struck piano without rhythmic pulse |
| `felt piano` | Muted, intimate piano — less attack, more pad-like |
| `bowed cello pad` | Sustained low string texture, no vibrato, no attack |
| `sustained strings` | Orchestral string bed, slow bow, meditative |
| `sub-bass drone` | Deep, grounding low-frequency hum |
| `shakuhachi flute` | Breathy Japanese bamboo flute textures, long decays |
| `glass harmonica` | Ethereal, high-frequency glass resonance |
| `gong` | Single distant strike with long reverb tail |
| `singing bowl strike` | Struck bowl with slow fade |
| `soft piano chords` | Slow, widely spaced chord voicings |
| `ambient guitar` | Processed, sustained guitar without rhythmic movement |
| `warm hum` | Vocal-adjacent drone, no words |
| `healing frequencies` | Warm, pure-tone harmonic content |

### What to include: Texture Descriptors

| Term | Effect |
|------|--------|
| `slow evolving` | Gradual tonal shifts over the full duration |
| `spacious` | Wide sense of space and air around the sound |
| `deep` | Emphasizes lower-frequency resonance |
| `warm` | Lower-mid warmth — works with the +1.5 dB shelf |
| `ethereal` | Light, floating, slightly otherworldly quality |
| `sustained` | Long note holds, minimal attack |
| `minimal` | Sparse elements, silence between sounds |
| `harmonic` | Overtone-rich resonance |
| `dark` | Darker tonal balance — more bass, less treble |
| `floating` | Sense of weightlessness, no rhythmic anchor |
| `layered pads` | Multiple pad textures stacked for depth |
| `very slow attack` | Notes fade in gradually — appropriate for drones |

### What to include: Atmospheric Anchors (optional, since base tags are already applied)

| Term | Effect |
|------|--------|
| `healing` | Restorative, welcoming warmth |
| `meditative` | Slightly ceremonial depth, intentional movement |
| `sleep` | Darker, slower, more hypnotic direction |
| `grounding` | Earth-toned, stable, lower-register emphasis |
| `introspective` | Quieter, inward quality, less motion |
| `sacred` | Reverent, ceremonial feel |
| `vast` | Expansive, almost empty space |

## Anti-Patterns — What NOT to Include

These inputs consistently produce output that does not suit meditation or degrades audio quality:

| Anti-Pattern | Why It Fails |
|---|---|
| `upbeat`, `energetic`, `dance`, `lively` | The LM planner interprets these as rhythmic intent — triggers pulse structures even without explicit BPM |
| `no melody`, `no chord changes`, `static harmony` | Contradictory directives confuse the LM planner — produces structurally incoherent output |
| `[Drone]`, `[Static Pad]`, `[Beatless]` in lyrics | Non-training-vocabulary tags are ignored or cause parsing issues; use only `[Instrumental]`, `[Intro]`, `[Verse]`, `[Bridge]`, `[Interlude]`, `[Outro]` |
| BPM numbers in caption (`60 BPM`, `72 BPM`) | BPM belongs in the **BPM slider only** — in the caption it confuses the text side of planning |
| Key names in caption (`C Major`, `D Minor`) | Key belongs in the **Musical Key dropdown only** |
| `binaural beats`, `432 Hz`, `528 Hz` | The LM planner has no mechanism to produce specific Hz tuning or binaural encoding — wastes caption tokens |
| `nature sounds`, `rain`, `ocean waves` | Can be produced but very unreliably; output quality is inconsistent |
| Multiple contrasting styles | `"energetic morning pads AND deep sleep drone"` — the LM planner cannot plan a single coherent arc from contradictory brief; choose one direction |
| `choir`, `vocals`, `voice`, `singing` | Although `instrumental=True` is set, these terms pull the LM planner toward vocal harmonic patterns that fight the constraint |
| `jazz`, `pop`, `EDM`, `classical` | Genre labels introduce structural patterns (chord progressions, rhythms) incompatible with meditation ambient |
| `fast attack`, `staccato`, `pizzicato`, `plucked` | Attack-heavy transients are jarring when the music is ducked under narration and snap back into audibility |
| `upward swell`, `dynamic crescendo` | The music is normalized then compressed — dramatic dynamics are attenuated; the LM planner wastes planning budget on dynamics that don't survive post-processing |

## Story Mode — Multi-Stage Generation

Story mode generates each meditation stage as a **separate ACE-Step inference call**, then blends adjacent stages with a **6-second equal-power cosine crossfade**. This allows deliberate tonal evolution across the meditation arc.

Story mode is activated by passing `music_prompt_stages` from the pipeline (not exposed as a UI field in the current implementation, but available via API). Each stage is a `(prompt, duration_sec)` tuple; the engine enhances each stage prompt independently via `_enhance_prompt()`.

### Story mode prompt principles

- **Each stage prompt stands alone** — it is enhanced independently. Do not write stage 2's prompt assuming the LM planner remembers stage 1.
- **Tonal consistency across stages** requires overlapping vocabulary. If stage 1 uses `warm analog pads` and stage 2 switches to `cold metallic drones`, the crossfade will bridge a significant timbral gap — increase duration allowance for the crossfade region.
- **Progression should be gradual**: `centering → deepening → integration` rather than `alert → shocked → asleep`.
- **The crossfade is 6 seconds** — each stage needs at minimum 15 seconds to have meaningful non-crossfade content. Stages shorter than 30s will feel mostly transitional.

### Example story mode stages

```python
# Centering → Depth → Integration arc (10 min total)
[
    ("warm analog synth pads, soft breathing texture, centering", 120.0),          # 2 min
    ("deep sub-bass drone, dark tibetan bowls, very slow evolution", 360.0),       # 6 min
    ("warm pads returning, soft light expanding, gentle and spacious", 120.0),     # 2 min
]
```

## The Full Caption — What ACE-Step Actually Receives

For the default single-prompt case at medium duration (e.g. 5 min):

**Your input prompt:**
```
soft tibetan singing bowls, warm analog pads, vast spacious reverb, deep and grounding
```

**Caption that reaches ACE-Step:**
```
ambient, meditation, calm, peaceful, spacious, soft dynamics, gentle, soothing, high fidelity, studio quality, clean production, soft tibetan singing bowls, warm analog pads, vast spacious reverb, deep and grounding, no vocals, instrumental
```
*(Note: `warm` and `spacious` base tags are suppressed because they appear in the user prompt)*

**Lyrics that reach ACE-Step:**
```
[Instrumental]

[Intro - Soft ambient texture fading in from silence, establishing space]

[Verse - Primary harmonic landscape, slow and contemplative]

[Bridge - Subtle tonal shift, deeper warmth, new colors emerge]

[Verse - Return to main theme with gentle enrichment]

[Outro - Extended fade, all elements dissolving gently into silence]
```

## Output Validation & Retry Behavior

The engine automatically validates every generated audio output for:

| Check | Threshold | Action on failure |
|-------|-----------|-------------------|
| NaN/Inf values | Any present | Retry (up to 3 attempts) |
| Near-silence | RMS < -50 dBFS | Retry |
| Excessive clipping | > 5% samples at ±1.0 | Retry |
| Short output | < 50% of requested duration | Retry |

**If all 3 attempts fail**, the last failed output is still returned with an error logged. A complete generation failure is usually a sign of a malformed prompt or an inference timeout.

## Quick Reference

| Factor | Guidance |
|--------|----------|
| **Prompt length** | 10–30 words. The 4B model handles longer, more complex prompts better than previous versions, but keep it focused on a single tonal world. |
| **Structure** | `[Primary source] + [Texture] + [Atmosphere]` — 2–5 comma-separated clauses |
| **Do include** | Instrument names, texture adjectives, slow/evolving/spacious/sustained, atmosphere anchors, complex harmonic descriptors |
| **Do not include** | BPM numbers, key names, genre labels, contradictory directives (no melody), binaural/Hz specifics, vocal-adjacent terms |
| **Do not repeat** | calm, peaceful, ambient, gentle, warm — already in base tags (filtered only if exact word match) |
| **BPM** | Set via slider, not prompt. Default 50. Keep ≤ 65 for deep meditation |
| **Key** | Set via dropdown, not prompt. Auto recommended unless you have a specific reason |
| **Quality** | Always use Studio for final output |
| **Duration routing** | ≤ 90s = single call; > 90s = three-phase pipeline (both are transparent to the prompt) |
| **Reference audio** | For timbre/style guidance only — not melody conditioning |
| **Validation** | Engine retries up to 3x automatically on near-silence, clipping, or short output |
